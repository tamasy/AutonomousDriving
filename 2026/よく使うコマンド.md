```bash
cd data/aichallenge/2026/aichallenge-racingkart

# 新規実行 (AWSIM + Autoware を自動起動)
make optimize-lqr

# ブラウザ Dashboard つき
make optimize-lqr OPTUNA_ARGS="--dashboard"

# トライアル数・タイムアウト指定
make optimize-lqr OPTUNA_ARGS="--n-trials 30 --timeout 120"

# 途中から再開
make optimize-lqr OPTUNA_ARGS="--resume"

# デフォルト値に戻す (コンテナ不要)
make optimize-lqr OPTUNA_ARGS="--restore-defaults"

# ドメインIDを指定して実行
make optimize-lqr OPTUNA_ARGS="--domain-id 1" OPTUNA_ARGS="--monitor --dashboard"

# モニタ付きで最適化開始
make optimize-lqr OPTUNA_ARGS="--monitor"
# → http://localhost:8090 をブラウザで開く（10秒自動更新）

# Optuna Dashboard も同時起動
make optimize-lqr OPTUNA_ARGS="--monitor --dashboard"

```

**動作フロー（各トライアル）:**
1. Optunaがパラメータを提案 → `reference.launch.xml` に書き込み
2. シミュレーション実行（`run_evaluation.bash`）
3. 終了条件（いずれか早い方）:
   - `run_evaluation.bash` が正常終了（1周以上完走）
   - スタック検知: 速度が閾値以下で一定時間停止
   - 上限タイムアウト: `optuna_trial_timeout` 秒経過
   - **所定周回到達**: `optuna_target_laps` 周完走（デフォルト: 1周）
4. `result-summary.json` の `min_time` を評価値として記録
   - 完走できなかった場合は 9999.0（ペナルティ）
5. Optunaが前回結果を参考に次のパラメータを決定（ベイズ最適化）

> 最初のトライアルは必ずデフォルト値から開始（ベースライン確立）
> 全トライアル終了後、最良パラメータが自動的に `reference.launch.xml` に書き戻される

**⚠️ スタック時の評価値は「1周完走前か後か」だけで決まる:**

スタック検知時、AWSIMにSIGTERM → `OnApplicationQuit()` → **完走済みラップ分のみ** `result-details.json` に書き出される。

| スタック地点 | 完走ラップ数 | 評価値 | Optunaへの影響 |
|------|------|------|------|
| 0.1周時点 | 0周 | **9999.0**（ペナルティ） | 悪いパラメータとして次回以降避けられる |
| 0.3周時点 | 0周 | **9999.0**（ペナルティ） | 同上 |
| 0.7周時点 | 0周 | **9999.0**（ペナルティ） | 同上 |
| 0.9周時点 | 0周 | **9999.0**（ペナルティ） | 同上 |
| 1周完走後にスタック | 1周 | **実際のmin_time** | 有効な結果として探索に活用 |

→ 境界は「1周完走」の前後のみ。0.9周でスタックしても開始直後と評価値は同じ。

**`--resume`（途中から再開）:**
- なし → 既存スタディを削除して新規作成（過去の結果を破棄）
- あり → SQLite DBから履歴を読み込んで追加トライアルを実行
- 例: 30トライアル指定で20回後にCtrl+Cした場合、`--resume` で残り10トライアルを続行可能

**`--target-laps N`（終了周回数の指定）:**
- 省略時は `reference.launch.xml` の `optuna_target_laps`（デフォルト: 1）を使用
- `--target-laps 0` で無制限（従来動作に戻す）
- 仕組み: `/localization/kinematic_state` をポーリングしてスタート位置からの距離を監視
  - スタート位置から 50m 以上離れた後、20m 以内に戻ったら 1 周カウント
  - 所定周回到達で AWSIM に SIGTERM → `OnApplicationQuit()` → `result-details.json` 取得

**`--restore-defaults`（デフォルト値に戻す）:**
- 最適化中に書き換わった `reference.launch.xml` をデフォルト値に戻す
- あわせて `control_mode` を `mpc` に復元（最適化中は `rule_based` に切り替わるため）

**最良結果の保管場所（2箇所）:**
| 場所 | 内容 | タイミング |
|------|------|----------|
| `/aichallenge/workspace/pure_pursuit_study.db`（SQLite） | 全トライアルの結果・パラメータ・スコアすべて | 各トライアル完了時にリアルタイム保存 |
| `reference.launch.xml` | 最良パラメータのみ | 全トライアル終了時に自動書き込み |
