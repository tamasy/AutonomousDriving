# update_quaternions — クォータニオン自動再計算ツール

PurePursuit形式の軌道CSVに対して、x/y座標から進行方向を計算し、クォータニオン（qx/qy/qz/qw）を自動で上書きする。

---

## 考え方

### なぜ作ったか

軌道CSVの座標（x/y）を手動で編集すると、クォータニオンが座標と食い違った状態になる。
クォータニオンを手計算で合わせるのは面倒で、ミスも起きやすい。

そこで **「x/yを渡すだけでクォータニオンを正しく再計算する」** ツールを作った。
座標を直接編集した後にこのスクリプトを1回叩くだけで、整合性を保ったCSVが得られる。

### 設計方針

- **上書き更新がデフォルト**: `-o` を省略すると入力ファイルを直接書き換える。変換のたびにファイルが増えない。
- **ループ軌道に対応**: 末尾の点は「末尾→先頭」方向でヨー角を計算するため、周回コースで先頭付近の向きが不連続にならない。
- **他カラムは保持**: `speed` など既存の他カラムは一切変更せず、qx/qy/qz/qw だけを更新する。

## 使い方

```bash
cd ~/data/aichallenge/2026/aichallenge-racingkart/aichallenge/tools

python3 update_quaternions.py <input.csv>               # 入力ファイルを上書き
python3 update_quaternions.py <input.csv> -o <output.csv>  # 別ファイルに出力
```

ファイル名のみ（パスなし）で渡した場合、`simple_trajectory_generator/data/` を自動参照する。

---

## 入力フォーマット

PurePursuit形式の軌道CSV（以下のカラムが必須）。

| カラム | 内容 |
|---|---|
| `x` | X座標 [m] |
| `y` | Y座標 [m] |
| `qx` | クォータニオン x（更新対象） |
| `qy` | クォータニオン y（更新対象） |
| `qz` | クォータニオン z（更新対象） |
| `qw` | クォータニオン w（更新対象） |

`z`, `speed` など他のカラムがあっても変更されずそのまま出力される。

---

## クォータニオンの計算ロジック

各点のヨー角を「現在点 → 次点」の方向ベクトルから求め、Z軸周りの回転クォータニオンに変換する。

```
dx = x[i+1] - x[i]
dy = y[i+1] - y[i]
yaw = atan2(dy, dx)

qx = 0.0
qy = 0.0
qz = sin(yaw / 2)
qw = cos(yaw / 2)
```

末尾点（`i = n-1`）は次点として先頭点（`i = 0`）を使用する（ループ閉じ）。

---

## アピールポイント

### 1. 座標を直接編集した後の「整合性修復」が1コマンドで完了

CSVのx/yを手で弄るたびにクォータニオンを手計算し直す必要がなくなる。
編集後に1回叩くだけで全点のクォータニオンが正しく揃う。

### 2. 上書きモードで余計なファイルが増えない

`-o` を省略すると入力ファイルを直接更新する。
「元ファイルのどれが最新か」で迷わないシンプルなワークフローになる。

### 3. ループ軌道の末尾も正しく処理する

末尾点の次点として先頭点を使うため、周回コースで末尾付近のヨー角が突然反転するような問題が起きない。

### 4. 先頭3行の即時確認

実行後に先頭3行の x / y / qz / qw が表示されるため、計算結果の妥当性をその場で目視確認できる。

```bash
# simple_trajectory_generator/data/raceline_awsim.csv を直接更新
```

## トラブルシューティング

| 症状 | 原因 | 対処 |
|---|---|---|
| `エラー: ファイルが見つかりません` | 入力ファイルが存在しない | パスを確認、またはデフォルトディレクトリに配置 |
| `エラー: 列 'qx' がCSVに存在しません` | MPC形式など非対応フォーマット | PurePursuit形式（x/y/qx/qy/qz/qw/speed）のCSVを使用する |
| `エラー: CSVにデータがありません` | ヘッダのみのCSV | データ行を確認 |
| 末尾付近のヨー角が不自然 | 閉じていない軌道（スタート≠ゴール）に適用した | ループ閉じの挙動を考慮して末尾の数点を手動調整 |
